# eclipse java doc & error

날짜: 2024년 6월 18일

```java
이클립스에서 export->java doc 를 생성했는데 안나오던 값이 
재 실행 했더니 나오는데 이유가 뭘까?
```

```java
이클립스에서 Javadoc을 생성할 때 특정 값이 처음에 나오지 않다가 재실행 시 나오게 되는 이유는
여러 가지가 있을 수 있습니다. 아래는 몇 가지 일반적인 원인과 그 해결책입니다:

    1. **빌드 문제**:
    - 처음에 Javadoc을 생성할 때 프로젝트가 완전히 빌드되지 않았을 수 있습니다. 
    이클립스에서는 소스 코드가 변경될 때마다 자동으로 빌드가 수행되지만, 
    가끔씩 빌드 과정에서 문제가 발생할 수 있습니다.
    - 해결책: 프로젝트를 수동으로 빌드합니다. "Project" 메뉴에서 "Clean"을 선택한 후 
    다시 "Build Project"를 선택합니다.
    
    2. **소스 코드 변경**:
    - Javadoc을 처음 생성했을 때 소스 코드에 변경 사항이 있었고, 
    해당 변경 사항이 저장되지 않았을 수 있습니다. 
    재실행 시에는 변경 사항이 저장되어 정상적으로 Javadoc이 생성될 수 있습니다.
    - 해결책: 소스 코드를 변경한 후 항상 저장하고 나서 Javadoc을 생성합니다.
    3. **캐싱 문제**:
    - 이클립스는 다양한 캐시 메커니즘을 사용합니다. 처음에 Javadoc을 생성할 때 
    캐시 문제가 발생하여 제대로 생성되지 않을 수 있습니다. 
    재실행 시 캐시가 갱신되어 문제가 해결될 수 있습니다.
    - 해결책: 이클립스를 재시작하거나, 워크스페이스를 갱신합니다. 
    워크스페이스를 갱신하려면 "File" 메뉴에서 "Refresh"를 선택합니다.
    
    4. **설정 문제**:
    - Javadoc 생성 시 설정이 제대로 되지 않았을 수 있습니다.
     예를 들어, 특정 패키지나 클래스가 Javadoc 생성 대상에서 제외되었을 수 있습니다.
    - 해결책: Javadoc 생성 설정을 확인합니다. 
    "Export" -> "Javadoc" 단계에서 설정을 꼼꼼히 확인하고 필요한 부분이 누락되지 않았는지 
    확인합니다.
    
    5. **이클립스 버그**:
    - 이클립스의 특정 버전에서 Javadoc 생성 시 버그가 있을 수 있습니다. 
    이 경우 이클립스를 최신 버전으로 업데이트하는 것이 좋습니다.
    - 해결책: 이클립스를 최신 버전으로 업데이트하거나, 다른 JDK 버전을 사용해 봅니다.
    위의 방법들을 통해 문제를 해결할 수 있을 것입니다. 
    문제를 해결한 후에도 동일한 문제가 발생한다면, 보다 구체적인 오류 메시지나 로그를 확인하여
    추가적인 원인을 분석하는 것이 필요할 수 있습니다.
```

`결론은 추정할 때 캐싱 문제 인 것 같다.`
