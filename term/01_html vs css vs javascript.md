# html vs css vs javascript

### HTML (HyperText Markup Language)

```jsx
HTML은 웹 페이지를 만들기 위한 기본적인 구조를 정의하는 마크업 언어입니다. 
HTML은 다음과 같은 기능을 수행합니다:
- **구조 정의**: HTML은 웹 페이지의 구조를 정의합니다.
이는 제목, 문단, 목록, 링크, 이미지 등과 같은 요소를 사용하여 정보를 구조화합니다.
- **콘텐츠 표시**: HTML은 텍스트, 이미지, 동영상 등 다양한 유형의 콘텐츠를 표시할 수 있습니다.
- **하이퍼링크**: HTML은 다른 웹 페이지로 연결되는 하이퍼링크를 생성할 수 있습니다.
```

### CSS (Cascading Style Sheets)

```jsx
CSS는 HTML로 작성된 콘텐츠의 시각적 표현을 꾸미는 스타일 시트 언어입니다. 
CSS는 다음과 같은 기능을 수행합니다:
- **스타일 적용**: CSS를 사용하여 웹 페이지의 요소에 스타일을 적용할 수 있습니다.
 이는 텍스트 색상, 글꼴, 배경색, 크기 및 위치 등을 조정하는 것을 포함합니다.
- **레이아웃 제어**: CSS를 사용하여 웹 페이지의 레이아웃을 제어할 수 있습니다.
 이는 요소의 위치, 크기 및 간격을 조정하여 페이지의 구조를 만드는 것을 포함합니다.
- **반응형 디자인**: CSS를 사용하여 웹 페이지를 다양한 디바이스 및 화면 크기에 맞게 조정하여
 반응형 디자인을 구현할 수 있습니다.
```

### JavaScript

```jsx
JavaScript는 웹 페이지의 동적인 기능을 추가하는 프로그래밍 언어입니다. 
JavaScript는 다음과 같은 기능을 수행합니다:
- **동적 콘텐츠**: JavaScript를 사용하여 웹 페이지에 동적으로 콘텐츠를 추가하거나 
변경할 수 있습니다. 이는 사용자 상호 작용, 데이터 로딩 및 콘텐츠 업데이트 등을 포함합니다.
- **이벤트 처리**: JavaScript를 사용하여 사용자 상호 작용에 대한 이벤트를 처리할 수 있습니다.
 예를 들어, 버튼 클릭, 마우스 이동 또는 키 입력과 같은 이벤트를 처리할 수 있습니다.
- **데이터 조작**: JavaScript를 사용하여 데이터를 조작하고 처리할 수 있습니다.
 이는 폼 데이터 유효성 검사, 배열 및 객체 조작, API 호출 및 데이터베이스와의 상호 작용 등을 
 포함합니다.

이것들은 각각 웹 개발에서 중요한 역할을 합니다.
  HTML은 웹 페이지의 구조를 정의하고, CSS는 해당 페이지의 디자인과 스타일을 정의하며,
  JavaScript는 페이지를 동적으로 만들고 상호 작용을 가능하게 합니다. 
  함께 사용되면 이들은 풍부하고 사용자 친화적인 웹 경험을 만들어냅니다.
```
